# H8-Fix: /api/termine/vorschlagen — User-Client statt Service-Role

> Erzeugt von Cowork QA, 18:40. Beta-Blocker.
> Direkt in Claude Code im Reparo-Ordner reinpasten.

## Problem

`POST /api/termine/vorschlagen` → 500 (Cowork-QA reproduzierbar). Insert kommt nie bis Postgres (0 Termine in `public.termine`). Root-Cause: `createServiceRoleClient()` throws weil `SUPABASE_SERVICE_ROLE_KEY` zur Build-Zeit nicht im Function-Bundle war (wurde nach dem letzten Build gesetzt). Auch nach Force-Redeploy (Commit e4a5c9a) bleibt 500 — vermutlich Function-Cold-Start-Caching.

## Lösung

`app/api/termine/vorschlagen/route.ts` umbauen: **statt `createServiceRoleClient()` den User-Client `supabase` nutzen** (kommt schon aus `getUserFromRequest`).

Begründung: `public.termine` hat bereits die nötigen RLS-Policies:

```
termine_insert: with_check (auth.uid() = handwerker_id)
termine_insert_beteiligte: with_check ((auth.uid() = handwerker_id) OR auth.uid() IN (ticket-Ersteller))
```

→ HW kann eigene Termine inserten ohne Service-Role-Bypass.

## Konkreter Diff

In `app/api/termine/vorschlagen/route.ts`:

1. **Zeile 75**: `const admin = createServiceRoleClient()` ENTFERNEN
2. **Zeile 76**: `await admin.from("tickets")...` → `await supabase.from("tickets")...`
3. **Zeile 88**: `await admin.from("einladungen")...` → `await supabase.from("einladungen")...`
4. **Zeile 115**: `await admin.from("termine").insert(rows)` → `await supabase.from("termine").insert(rows)`
5. **Zeile 119**: `await admin.from("profiles")...` → `await supabase.from("profiles")...`
6. **Zeile 124**: `await admin.from("profiles")...` → `await supabase.from("profiles")...`
7. **Import-Zeile oben**: `createServiceRoleClient` aus dem Import streichen (wird nicht mehr genutzt in dieser Datei).

**Wichtig:** Falls die `tickets.select()` für `erstellt_von` (Zeile 76-83) ohne RLS-Bypass nicht klappt (weil termine-vorschlagen-HW kein RLS-Recht auf tickets.erstellt_von hat), dann muss eine SELECT-Policy auf tickets ergänzt werden, oder die Mieter-Email-Suche per Edge-Function hinter Helper. Erst probieren, ob's so geht — wenn nicht, sag Bescheid.

**Commit:** `fix(termine): User-Client statt Service-Role (H8, ENV-cold-start-resistent)`

## Constraints

- Nur diese Route ändern, andere Service-Role-Routes (z.B. `/api/auftraege/annehmen`) bleiben wie sie sind (die haben anderen Use-Case)
- Email-Versand bleibt unverändert (fire-and-forget, schluckt Fehler)
- Nach Push: Cowork (im anderen Chat) wird automatisch K1.1+K1.2 re-testen
