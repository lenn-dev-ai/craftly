## Item 2: UX-Konsistenz — linke Admin-Sidebar ≠ Mobile-BottomNav (fee57a75)

**Lennart-Beobachtung:** „Warum ist der linke Stammbaum zur Auswahl mit anderen Punkten versehen als und die horizontale Auswahlleiste"

**Analyse:** Im Admin-Layout zeigt die linke Sidebar (Desktop) andere Menüpunkte als die untere BottomNav (Mobile). Sollte synchron sein, sonst verliert User Orientierung beim Geräte-Wechsel.

**Code-Lokation:**

- `components/layout/BottomNav.tsx` Zeile 38–43 (Admin-BottomNav-Items)

**Aktueller BottomNav-Admin-Stand:**
```ts

  { href: "/dashboard-admin",                 label: "Start",      Icon: LayoutDashboard },
  { href: "/dashboard-admin/nutzer",          label: "Nutzer",     Icon: Users },
  { href: "/dashboard-admin/aktivitaet",      label: "Aktivität",  Icon: Activity },
  { href: "/dashboard-admin/system",          label: "System",     Icon: Settings },
],
```

**Aktuelle Mobile-Erweiterung (P3.4):** Feedback wurde der BottomNav hinzugefügt — aber damit sind 5 Items in der Mobile-Leiste und das war eigentlich auf 4 begrenzt. Plus: Sidebar hat ggf. noch mehr Items.

1. Beide Listen abgleichen (Sidebar + BottomNav).
2. Mobile-BottomNav auf max. 5 Items: Start / Nutzer / **Feedback** / Aktivität / System (ggf. 1 davon raus).
3. Wenn Mobile 4-Item-Layout besser → entscheide welche 4 wichtigsten (Vorschlag: Start / Feedback / Nutzer / System).
4. Sidebar darf mehr Items haben (Desktop hat Platz), aber die in BottomNav müssen Subset davon sein.

**Commit-Erwartung:** `fix(admin-nav): Sidebar+BottomNav konsistent (UX/fee57a75)`. Wenn du Layout-Entscheidungen treffen musst (z.B. „welche 4 in Mobile?"), 1 Frage an Lennart, sonst implementieren.
